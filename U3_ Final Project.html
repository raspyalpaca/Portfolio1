    U3: Final Project

    • Due May 12 by 8am
    • Points 100
    • Submitting a website url
    • Available until May 12 at 8am

    The goal of this project is to encompass all the skills we learned so far. You will be creating a dynamic web site using PHP, MySQL, HTML and CSS. The total project is 100 points.

    Requirements

    1. Design – 20%

      Design your site around your content and audience. Within your “includes” folder, you must have at least a header.html, footer.html and mysqli_connect.php. Your pages will then include/require these pieces. Be sure to use an external style sheet and put any images in a folder called “images”. Your design will also be graded based on creativity and appeal. Don’t forget navigation so the user doesn’t need to use the back button.

    2. Database – 20%

      Your site (or pages of your site) must by dynamic in nature, meaning some of the content must come from a database. You will need to design your own database. The data must be relational and pulled from two or more tables. This does not have to be the case for every query, but there must be at least one query with a JOIN.

    3. Functionality – 20%

      The user should be able to interact with the content. This includes:

      1. Sorting columns
      2. Clicking on rows for detailed information

      The user should be able to manage the content in the database using forms. This includes:

      1. Adding content (INSERT)
      2. Editing content(UPDATE).
      3. Removing content (DELETE). There should be some type of confirmation for this. It can be JavaScript or PHP.

    4. Additional Functionality – 20%

      Pick one of the following to add to your site. (Extra Credit: Additional options can be added for an extra 10 points each.)

      1. Shopping Cart – implement functionality to add items to a cart that is stored in the session. Make sure to include a way to update quantities or remove items from the cart. Make sure to calculate the total for the cart and provide a way to“checkout” and clear the cart.
      2. Secure Editing – add logins to your site (sessions, registration, and login functionality) to prevent unauthorized editing of records. (See chapter 18)
      3. Rating System – implement functionality to allow users to rate or rank items. This could be accomplished with reviews or thumbs up/down system. These should be used to help the user make a decision.
      4. Pagination – add the ability to paginate through long lists of records. This should include the ability to view the next/previous page without errors (no previous button on first page and no next button on last page) and modify how many records are displayed per page. (See chapter 10)
      5. Ajax – use ajax to streamline functionalities of the site. This could be used for something like searching records or enhancing the adding of items to a cart.
      6. ??? – present an idea to your instructor to see if it qualifies. Approval must be granted before development starts.

    5. Validation and Sanitation – 20%

      All inputs must be validated before put into SQL statements. Required fields should not be empty. The error messages provided should help the user correct their error and the form should remain sticky. Your database should be protected against SQL injections using prepared statements and XSS attacks using a method of your choice.
